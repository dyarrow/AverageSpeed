Introduction

The Neology Average Speed Enforcement tool validates that average speed cameras are reading and recording vehicle speeds accurately. Calibrated VBox GPS data-loggers are driven through the enforcement section, and the average of the GPS-measured speeds are compared against the OBO calculations.

◉ Camera A
🚗
known distance
Exit
◉ Camera B

VBox measures

GPS-derived speed and precise timestamps (at 10 or 25hz for Vbox, 100hz for OxTS RT) throughout the enforcement section.

OBO records

The camera system's speed calculation based on the known inter-camera distance and passage timestamps.

i
The tool compares the average of the GPS points with the calculated OBO values and checks whether the difference falls within configured tolerance thresholds — giving a clear pass or fail for each passage.
01

Prepare test vehicles

Before driving the section, install a VBox unit in each test vehicle and note the vehicle's VRM (front and rear plates).

Install the VBox unit in the test vehicle

Mount the VBox data-logger with a clear view of the sky. Ensure the antenna has unobstructed GPS reception. Allow at least 5 minutes for satellite lock before beginning runs.

Note the vehicle VRM — front and rear plates

Record the vehicle registration mark for both the front and rear plates. Average speed cameras read plates in both directions, so the tool needs to know which VRM to match in the OBO data.

Example vehicle record

Front
AB12 CDE
Rear
AB12 CDE

Check time synchronisation

Ensure the VBox clock is synchronised to UTC. If your VBox records in local time or has a known drift, configure the Time offset value in the Settings dialog before running the comparison. The Leap seconds setting is only required for legacy VBox units such as the SX10.

02

Run vehicles through the section

Drive each test vehicle through the average speed enforcement section at representative speeds. The VBox records the GPS track throughout.

Drive through at consistent speeds

Enter the section at a steady speed.

Recommended runs

Minimum 10 passes per vehicle per direction at different speeds across the enforcement band.

Direction

Run vehicles in both directions if the site enforces both.

Note the time window of each run

Record the approximate start and end time of your run session. You will need this when exporting OBO data — you only need to export the period covering your test runs.

i
Leave a buffer of at least 5 minutes either side of your run window when exporting OBO data to ensure no passages are missed.
03

Collect VBox data

Once all runs are complete, retrieve the GPS data files from the VBox units in each vehicle.

Download from the VBox unit

Connect the VBox to a laptop via USB or remove the SD card. Copy the data files for each run session to a clearly labelled folder per vehicle.

Supported file formats

.vbo — VBox native .csv — OxTS format

The tool auto-detects the format from the file extension. A single vehicle may have multiple run files — all can be added together in the wizard.

04

Export OBO data from the camera system

The OBO file is the camera system's record of all vehicle passages. Export this for the time window covering your test runs.

Export from the OBO

Access the OBO (Operational Back Office) interface and export the passage data as an Excel file (.xlsx) for the date/time range covering your run session (plus the 5-minute buffer). The tool validates the file on import — if required columns or the Matched sheet are missing, a clear error message is shown.

OBO file requirements

✓  Excel format (.xlsx)
✓  Contains a sheet named Matched
✓  Includes columns: LP Number, LP Hash, Entry Primary Time, Entry Secondary Time, Exit Primary Time, Exit Secondary Time, Primary Speed, Secondary Speed, Primary Camera ID
!
Double-check export settings if the tool reports a validation error — the Matched sheet name and column layout must match exactly.
i
Plate hashes: If the OBO is configured to use hashed plates only, you will need to determine which hash matches each vehicle using the image review and enter this in the wizard instead of the VRM.

Use the column filter icons (funnel on each header — click the left edge of any column header) to filter the results table by VRM, speed range, or pass/fail status.
08

Pass/fail thresholds

Thresholds are configured in Settings (top-right of the main window). Two modes are available:

Percentage only mode

Pass if the speed difference is within ±N% of the VBox speed. Applied to all passages. Enabled by ticking Percentage only in Settings.

Speed breakpoint mode

Below the breakpoint: apply a ±MPH tolerance. Above the breakpoint: apply a ±% tolerance. Allows tighter absolute tolerances at lower speeds.

i
Changing thresholds in Settings immediately re-colours the existing results table — you do not need to re-run the comparison.

Troubleshooting

No passages matched for a vehicle

Check that the VRM entered in the wizard exactly matches what appears in the OBO file. Spaces, leading zeros, and letter case can all cause mismatches. If the site uses hashed plates, ensure the correct hash is entered.

GPS timestamps don't align with OBO times

Configure the Time offset value in Settings. The VBox clock must be in UTC — if it records local time, apply the appropriate hour offset. The Leap seconds setting is only required for legacy VBox units such as the SX10.

Passages found but speeds differ wildly

This usually means the wrong GPS file was assigned to a vehicle, or the VBox file covers a different run session than the OBO data. Check the timestamps in both files match the expected run window.

OBO file validation error

The tool checks for a sheet named Matched and required columns. If the export uses a different sheet name or column layout, check the export settings in the camera management system.

i
Restore Previous: After a successful wizard run, the vehicle configuration is saved automatically. Next time you open the wizard, click Restore Previous to reload all VRMs and GPS file paths without re-entering them.
